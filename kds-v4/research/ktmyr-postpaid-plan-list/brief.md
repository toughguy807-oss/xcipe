## 핵심 발견: 페이지의 실제 정체

⚠ **URL 경로는 `/fe/mypage/ppl/`** (마이페이지 하위) **이지만 실제로는 공개 후불요금제 상품 목록 페이지**. 로그인 없이 250개 후불요금제 카드가 모두 노출됨.

> 정확한 "마이페이지 안 가입한 후불요금제 목록" 은 로그인 필요로 접근 불가. **본 분석은 가장 유사한 정보 구조의 공개 페이지 (= 상품 후불요금제 목록) 기반**. 가입자 마이페이지 PPL 목록을 시안화하려면 본 페이지의 카드 정보 위계 (사업자 로고 + 이름 + 가격 + 스펙 + 액션) 를 베이스로 사용하고, "비교" 액션 자리를 "변경/해지/상세보기" 로 치환하는 변형이 자연스러움.

이 사실은 시안 디자이너에게 중요:
- 본 페이지의 카드 정보 위계는 **"상품 정보 + 비교 선택"** 중심
- 만약 시안이 진짜 마이페이지 후불요금제 목록 (이미 가입한 요금제) 이라면 정보 위계가 **"가입 상태 + 사용량 + 변경/해지 액션"** 중심으로 달라져야 함
- 본 brief 는 두 가능성 모두 고려해 정보·UX 후보를 제시

## 확인 필요

1. **마이페이지 안 실제 "가입한 후불요금제 목록" 페이지** — 로그인 필요로 직접 접근 불가. 사용자 자체 가입 정보 화면을 시안에 반영하려면:
   - 옵션 A: 본 brief 의 공개 페이지를 베이스로 하고 "비교 선택 → 변경/해지/상세보기 액션" 으로 치환
   - 옵션 B: 사용자가 로그인 후 직접 스크린샷 제공
   - **메인 세션이 사용자에게 의도 확인 필요**: 만들려는 화면이 "상품 카탈로그 (이번 페이지)" 인지 "가입자 자기 요금제 보기" 인지?

2. **데스크탑 시안 동시 작업 여부** — 반응형 사이트이므로 두 viewport 모두 시안화 가능. 현재 기본은 모바일이지만 사용자가 데스크탑 시안도 원하면 추가 호출 필요

3. **`#14BAD7` 의 KDS 정확 매핑** — KDS primitive 에 cyan 계열이 있는지 / teal.300 (#4cc8c3) 와 cyan.500 중 어느 쪽이 더 정합한지. **디자이너가 `kds/tokens/color.primitive.json` 을 보고 결정**. 만약 둘 다 부적합하면 `teal.500 #007f7f` 로 한 단계 채도 낮춰 매핑 (가격 강조의 시각적 무게는 KDS 토큰 안에서만 표현)

4. **필터 버튼 다크 네이비 `#373B59` 의 KDS 매핑** — 디자이너 검토. 후보:
   - `coolGray.700/800` 으로 매핑 (가장 가까움)
   - 또는 `fill.primary` 검정 (#191a1b) 으로 통일 (절제 강조)
