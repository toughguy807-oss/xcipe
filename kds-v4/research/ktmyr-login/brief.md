## 개요

- 목적: KT 마이알뜰폰 로그인 페이지의 정보 구조·UX 흐름·컴포넌트 구성 분석. KDS 재디자인 시안 3개 작업의 입력
- 시드 URL: `https://ktmyr.com/fe/mypage/com/comLoginForm.do`
- 모드: single (워크플로우 B B-1 b)
- 도메인: `ktmyr.com` (eTLD+1) — 브랜드 정보는 `research/brands/ktmyr.com/brand.md` 참조
- 수집 페이지: 1개
- 생성일: 2026-05-18
- 페이지 타이틀: `로그인 | KT 마이알뜰폰`

- **헤더에 GNB 가로 메뉴 추가**: 7개 카테고리 (마이알뜰폰 / 상품 / 유심·eSIM / 이벤트 / 고객지원 / 서비스) — 모바일은 햄버거 메뉴로 압축

## 동적 상태 / 인터랙션 흐름 (관찰·추정)

> Playwright 캡처는 **로딩 직후 정적 화면**만 잡힘. 아래는 DOM 의 hidden input·script 변수로 추정한 동적 흐름.

### 로그인 시도 시 흐름 (form action 분석)

- form action `https://ktmyr.com/fe/mypage/com/comLoginForm.do` / method POST
- hidden input 으로 `publicKeyModulus`, `publicKeyExponent` — **RSA 클라이언트 사이드 암호화** (rsaObj 변수)
- hidden `recaptchaSkipYn`, `recaptchaKey` 와 `g-recaptcha-response` textarea — **reCAPTCHA v2 invisible** 적용 (캡처에서도 우하단 reCAPTCHA 배지 보임)
- `netfunnelActionId` — **NetFunnel 대기열 시스템** (KT 그룹 공통, 트래픽 폭주 시 대기 화면 적용)
- 추정 실패 흐름: 인증 실패 → 동일 페이지 reload + 에러 메시지 (캡처 미확보)

### 2차 인증 흐름 (hidden DOM 흔적)

- hidden 입력들: `athuMbrPhone` (회원정보 등록 전화번호), `btnAuthTel` (인증요청 버튼), `btnReAuthTel` (재요청), `otpNum` (인증번호), `intOtpYn`, `intOtpCertYn`
- placeholder (원문): `회원정보에 등록된 전화번호를 입력해주세요`, `인증번호를 입력해주세요`
- 즉 비밀번호 5회 오류 / 신규기기 로그인 / 본인확인 필요 시 **휴대폰 SMS OTP 인증** 단계로 진입하는 모달/별도 화면 존재
- 버튼 디자인: bg `rgb(240, 240, 240)` (라이트 회색) / border `rgb(73, 150, 144)` (Teal) / radius 4 — 본문 버튼과 다른 outline 스타일 (서브 단계 강조 낮춤)

### 아이디 찾기 / 비밀번호 찾기

- form action `https://ktmyr.com/fe/mypage/find.do` / method GET — 별도 페이지로 이동
- 본 페이지에서는 링크만 노출

### 비번 토글 (눈 아이콘)

- 클래스 `btn_pw_toggle btnPwOff` → 클릭 시 `btnPwOn` 으로 토글되며 PW input type 변경 추정

### 자동로그인 / 아이디 저장

- 체크박스 `saveId` 만 노출 — **자동로그인 옵션 없음** (보안 정책 — KT 본사도 동일 패턴)

## UX 라이팅 톤 (관찰 — 원문 보존)

- **존대 + 권유체**: "~해주세요", "~해보세요". 명령형 아님.
- **마침표 사용 일관**: "아이디를 입력해주세요.", "비밀번호를 입력해주세요." 끝에 마침표.
- **친근한 1인칭 거절 문구**: "괜찮아요. 모바일 웹으로 볼게요." — dismiss 문구를 사용자 입장에서 작성한 점 특이.
- **느낌표 빈도**: 권유 카피 ("앱 다운로드", "관리하세요!", "한번에!") 에 종종 사용. 폼 자체 카피에는 없음.
- **명사형 종결 강조**: "무제한 혜택 제공", "데이터 쿠폰 제공" 등 혜택 라벨은 동사 없이 짧게.
