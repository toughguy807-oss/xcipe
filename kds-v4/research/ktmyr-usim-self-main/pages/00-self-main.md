## 인터랙션 흐름 (관찰)

1. 사용자는 LNB 칩 `셀프개통` 진입 → 페이지 sub-tab 으로 `셀프개통` / `개통신청조회` 분기
2. Hero 캐러셀로 가치제안 2개 노출 (개통 자체의 쉬움 + 회선 등록의 편익)
3. 정보 박스에서 즉시 시작 가능한지 시간 확인 → **메인 CTA `5분 만에 셀프개통 하기`** 클릭이 핵심 컨버전 동선
4. 사전 준비 부족 (유심 없음) → 유심 구매 옵션 2개 (배송/편의점) 로 분기
5. 사용자가 절차 학습 필요 → 6 step 가이드 → 각 step 자세히 보기
6. 사용자가 요금제도 같이 결정 → 추천 요금제 swiper → `상세보기` / `가입하기` (가입 CTA 가 자체 셀프개통 흐름과 연결)
7. 의구심 발생 → FAQ 아코디언 5개
8. 잔존 의문 → 유의사항 + eSIM 별도 안내 링크
9. 챗봇 FAB 가 페이지 어디서나 진입 가능 (보조)

**컨버전 동선의 양축**: ① 메인 CTA `5분 만에 셀프개통 하기` (Hero 영역 직속) / ② 요금제 카드의 `가입하기` (제품 단위 진입)

## Notes (에이전트 해석)

- **이 페이지는 "정보 전달 + 컨버전 유도" 가 균형. 단순 가입 페이지가 아니라 학습 자원 (6 step 가이드, FAQ, 유의사항) 이 큰 비중 차지.** 사용자의 "할 줄 모를까봐 걱정" 을 정면 해소하는 페이지. 모바일이면 가이드·FAQ 가 페이지 절반 이상.
- **brand 의 양축 컬러 정책 일관 적용**: Purple (`#6B3BE9`) 이 모든 메인 CTA + step 인디케이터에 동일 사용. Teal 은 여기 화면에서 거의 등장 안 함 (헤더 로고의 "알" 한 글자만) — 셀프개통은 **Purple Primary** 영역.
- **6 step 인디케이터 패턴이 특징적**: 보라 fill + 라일락 보더 4px = "이 색의 강조는 KT 마이알뜰폰의 핵심 시각 시그니처". KDS 의 `step indicator` 패턴으로 매핑 가능 (KDS 에 step indicator 컴포넌트가 있는지 확인 필요).
- **FAQ 가 클릭형 아코디언 vs 단순 카드 리스트** — 모바일에서 펼침 상태를 못 봤지만 button 태그 + chevron 추정으로 아코디언으로 본다.
- **요금제 카드의 `가입하기` 가 자기 자신 셀프개통 흐름과 연결됨**: 추천 요금제 자체가 셀프개통 절차의 STEP 02 (요금제 선택) 와 동일 동선 — 페이지 내부에서 미리보기 + 단축 진입을 동시에 제공.
- **컴포넌트 폭 다름 주의**: 가이드 섹션은 1024 wide, FAQ 도 1024, 다른 섹션은 1200 wide — 데스크탑에서 정렬 폭이 일관되지 않음. KDS 표준은 main content 폭 일관 유지.
- 모바일에서 hero 캐러셀이 **가로 풀폭 - 좌우 padding 20** 형태. 정보 박스도 동일 폭 — 1열 stack.
