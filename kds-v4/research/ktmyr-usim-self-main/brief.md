## UX 흐름

1. 진입 → hero 슬라이드 인지 (개통 가치 / 회선 관리 가치)
2. 가능시간 확인 → 지금 진행 가능 여부 판단
3. 유심 보유 자가진단 → 미보유 시 우회 경로 (바로배송/편의점)
4. 가이드 step 정독 → 셀프개통 절차 학습
5. CTA 클릭 → 비로그인이면 게스트 정보 입력 팝업, 로그인이면 usimSelfReqView.do 로 진입

## 시안 변형 적용 (이번 작업)

히어로/안내 영역이 차별화 주축임 → 세 시안 모두 hero + 가능시간 카드 + 가이드 시작점의 배치/톤/UX 를 적극 변형.

- 시안 A (레이아웃): 좌우 분할 hero (PC) / 가능시간 카드를 hero 위로 끌어올린 스택 (MO) — 정보 우선 노출
- 시안 B (톤): 다크 hero + 강한 teal accent + 큰 헤드라인 타이포 — 임팩트 우선
- 시안 C (UX): 사전 준비 체크리스트 우선 (유심·본인인증·신분증·납부수단 4개 체크) → 모두 OK 면 CTA 활성, 미체크면 우회 경로 안내 — task 완수 보조 우선

모든 시안에서 가이드 6 step / 추천 요금제 / FAQ 섹션은 베이스 톤 유지 (변형 안 함). hero/안내 영역만 적극 변형.
