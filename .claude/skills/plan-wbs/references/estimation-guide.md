## 2. 3점 추정법 적용 예시

```
낙관(O) = 3일, 최빈(M) = 5일, 비관(P) = 10일
기대값 E = (O + 4M + P) / 6 = (3 + 20 + 10) / 6 = 5.5일
표준편차 σ = (P - O) / 6 = (10 - 3) / 6 = 1.17일
```

**신뢰구간**:
| 범위 | 확률 | 값 |
|------|------|-----|
| E ± 1σ | 68% | 4.3 ~ 6.7일 |
| E ± 2σ | 95% | 3.2 ~ 7.8일 |
| E ± 3σ | 99.7% | 2.0 ~ 9.0일 |

> 고객 보고용 = E + 1σ (68% 신뢰). 내부 관리용 = E (기대값).

### 적용 예시: 공공기관 중간 복잡도 페이지 10개

```
기본 산정:
  기획: 10 × 1인일 = 10인일
  디자인: 10 × 2인일 = 20인일
  퍼블리싱: 10 × 1.5인일 = 15인일
  개발: 10 × 2인일 = 20인일
  QA: 10 × 1인일 = 10인일
  소계: 75인일

보정 적용 (공공기관):
  기획: 10 × 1.3 = 13인일 (문서화·결재 프로세스)
  디자인: 20 × 1.0 = 20인일
  퍼블리싱: 15 × 1.2 = 18인일 (접근성 강화)
  개발: 20 × 1.1 = 22인일 (보안·인증 강화)
  QA: 10 × 1.5 = 15인일 (접근성·보안 검증)
  소계: 88인일

버퍼 (15% 기본):
  88 × 1.15 = 101.2 → 102인일

최종 산정: 102인일 (약 20.4주, 1인 기준)
```

## 4. 버퍼 적용 기준

| 상황 | 버퍼율 | 근거 |
|------|--------|------|
| 요구사항 확정, 기존 기술 스택 | 10% | 낮은 불확실성 |
| 요구사항 80% 확정, 일부 신기술 | 15% | 기본 |
| 요구사항 미확정 항목 존재 | 20% | 범위 변경 가능성 |
| 고객 의사결정 지연 예상 | +5% | 대기 시간 보정 |
| 신기술 도입 (팀 경험 없음) | +30% | 학습 곡선 |
| 레거시 연동 (문서 부족) | +20% | 리버스 엔지니어링 |
| 다수 이해관계자 (5명+) | +10% | 커뮤니케이션 오버헤드 |

### 버퍼 중첩 규칙

버퍼는 **최대 2개까지 합산**, 이후는 최대값만 적용:

```
예: 신기술(+30%) + 레거시 연동(+20%) + 다수 이해관계자(+10%)
→ 상위 2개 합산: 30% + 20% = 50%
→ 10%는 미적용 (과다 산정 방지)
```

## 5. Wave 분류 실무 가이드

### Wave 판단 기준

| Wave | 기준 | 예시 |
|------|------|------|
| W1 (즉시) | 선행 의존 없음 + 리소스 확보 | 정적 페이지, 단순 CRUD |
| W2 (후속) | W1 완료 후 가능 또는 외부 의존 | API 연동, 데이터 마이그레이션 |
| W3+ (최종) | W1+W2 완료 후 통합 | E2E 테스트, 성능 최적화, 배포 |

### Wave 배분 비율 가이드

| 프로젝트 규모 | W1 | W2 | W3+ |
|-------------|-----|-----|------|
| 소규모 (≤20 FN) | 60% | 30% | 10% |
| 중규모 (21~50 FN) | 50% | 35% | 15% |
| 대규모 (50+ FN) | 40% | 40% | 20% |

## 7. 마일스톤 설정 가이드

### 표준 마일스톤 5단계

| # | 마일스톤 | 시점 | 산출물 |
|---|---------|------|--------|
| M1 | 기획 완료 | 전체 10~15% | QST, REQ, FN 확정 |
| M2 | 디자인 확정 | 전체 25~30% | 스타일 가이드, UI 명세 |
| M3 | 퍼블리싱 완료 | 전체 50~60% | HTML/CSS/JS 산출물 |
| M4 | QA 완료 | 전체 80~85% | 테스트 리포트, 이슈 0건 |
| M5 | 오픈 | 전체 100% | 라이브 배포, 인수인계 |

### 마일스톤 간 의존성

```
M1 ──→ M2 ──→ M3 ──→ M4 ──→ M5
 │              │       │
 └── 기획 변경 시 M2~M5 재산정
                └── 퍼블리싱 지연 시 M4 조정
                        └── QA 블로커 시 M5 연기
```
