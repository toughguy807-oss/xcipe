# Layout 설계 (Design-Layout) 체크리스트

## 페이지 커버리지
- [ ] IA의 모든 페이지에 레이아웃 패턴(A~F) 배정
- [ ] 누락 페이지 0건
- [ ] 각 페이지에 패턴 선택 사유 기록

## 공통 구조
- [ ] Header/GNB 공통 정의 (로고 위치, 메뉴 수, 유틸)
- [ ] Footer 공통 정의 (사이트맵, 법적 고지, SNS)
- [ ] Container max-width + padding 정의 (M/T/D별)

## 섹션 분해
- [ ] 각 페이지가 S1~Sn 섹션으로 분해
- [ ] 미분해 페이지 0건
- [ ] 각 섹션에 패턴 코드 명시

## 반응형 3단계
- [ ] Mobile (≤768px) 레이아웃 정의
- [ ] Tablet (769~1024px) 레이아웃 정의
- [ ] Desktop (≥1025px) 레이아웃 정의
- [ ] 스킵된 뷰포트 0건
- [ ] 브레이크포인트별 그리드 변화 명시

## STYLE 토큰 참조
- [ ] 간격이 `--spacing-*` 토큰 참조
- [ ] 컨테이너가 `--container-*` 토큰 참조
- [ ] radius가 `--radius-*` 토큰 참조
- [ ] 하드코딩된 수치 0건

## 시각 리듬
- [ ] 연속 동일 패턴 회피 (같은 패턴 3연속 이하)
- [ ] 페이지 간 레이아웃 일관성 유지
- [ ] 콘텐츠 양 변동 시 레이아웃 안정성 확인

## 모바일 사용성
- [ ] 터치 타겟 44×44px 이상
- [ ] 햄버거 메뉴 또는 적절한 모바일 네비게이션
- [ ] 가로 스크롤 발생 구간 없음
