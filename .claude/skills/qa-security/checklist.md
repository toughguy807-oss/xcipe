# qa-security 검수 체크리스트

reviewer 검수 시 사용합니다.

## 구조 완전성
- [ ] 프로젝트 스택 자동 감지 완료 (JSP/Laravel/Vue/Filament/공통)
- [ ] 감지된 스택에 해당하는 룰셋 전수 적용 (SEC-F/J/L/V/FL/S)
- [ ] 취약점 현황 요약 테이블 존재 (Critical/Major/Minor/Info)
- [ ] 스택별 결과 분리 출력
- [ ] 주요 취약점 상세 섹션 존재

## 품질 기준
- [ ] 모든 취약점에 파일:라인 위치 명시
- [ ] 모든 Critical/Major에 구체적 수정 코드 기재
- [ ] 오탐 필터링 수행 (정적 문자열/테스트 파일/주석 내 코드 제외)
- [ ] 통과 기준 적용 (Critical 0건 + Major 0건 또는 수용 사유)
- [ ] OWASP Top 10 카테고리 매핑

## 금지 사항
- [ ] 파일 경로 없이 취약점 보고 금지
- [ ] 수정 가이드 없는 Critical/Major 보고 금지
- [ ] 스택 미감지 상태에서 스캔 진행 금지
- [ ] 오탐 미필터링 상태로 결과 출력 금지
