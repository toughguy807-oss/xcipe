# REQ 체크리스트

## 필수 항목
- [ ] QST 기반 자동 반영 확인 `[연계 모드 전용]`
- [ ] FR/NFR 분류 완료

## FR (기능 요구사항)
- [ ] 모든 FR에 고유 ID (FR-###)
- [ ] 모든 FR에 설명 + 배경 (무엇/왜/누구)
- [ ] 모든 FR에 구현 명세 (Specifications)
- [ ] 모든 FR에 비즈니스 규칙 (조건 -> 동작)
- [ ] 모든 FR에 완료 기준 (AC) 1개 이상
- [ ] 모든 FR에 메타데이터 (우선순위/공수/의존성/담당/출처/분류근거)

## NFR (비기능 요구사항)
- [ ] 모든 NFR에 측정 기준 + 목표값
- [ ] 모든 NFR에 임계값 + 대응 방안

## 우선순위
- [ ] MoSCoW 분류 (Must/Should/Could/Won't)
- [ ] Impact x Effort 매트릭스 적용
- [ ] 분류 근거 1줄 이상

## 추적성
- [ ] Q-### -> FR-### 매핑 일관성 `[연계 모드 전용]`
- [ ] 추적성 매트릭스 존재 `[연계 모드 전용]`
- [ ] [미확인] 0건 확인

## 범위 경계 (구축/전환 모드)
- [ ] 5개 영역 모두 최소 1개 이상 항목 정의
- [ ] Negotiable 항목에 조건/대안 기재
- [ ] 총 항목 수 15개 이상

## CR 변경 관리 (운영 모드)
- [ ] CR 라이프사이클 6단계 존재 (접수→영향분석→승인→반영→검증→완료)
- [ ] CR 템플릿 8필드 완비 (ID/요청일/요청자/유형/범위/등급/공수/승인상태)
- [ ] 영향 분석 6항목 기재 (기능/일정/비용/품질/리스크/이해관계자)
- [ ] 스코프 크립 경보 (20% 초과 경고) 규칙 명시

## 운영 모드
- [ ] RCA 4필드 (현상/원인/영향/해결방향) 완비
- [ ] 모든 변경 FR에 변경 유형 태그 ([수정]/[추가]/[삭제])
- [ ] 모든 변경 FR에 AS-IS / TO-BE 기술
- [ ] 모든 변경 FR에 영향 범위 (연관 FR/FN) 명시
- [ ] 모든 변경 FR에 롤백 조건 기술
- [ ] 단계별 해결 계획 (즉시/단기/모니터링)
- [ ] REQ 수준 롤백 계획 4필드 (트리거/방법/영향/소요)
