# 업종별 질의 참고 가이드

> plan-qst 스킬 참조용. SKILL.md §업종별 특화 질의를 보완하는 실무 질의 패턴.

---

## 1. 업종별 심화 질의 패턴

### 이커머스 / 쇼핑몰

**결제·주문**:
- 결제 수단별 PG사 연동 현황? (카드/계좌이체/간편결제/해외결제)
- 부분 취소·환불 프로세스? (PG 연동 vs 수동 처리)
- 주문 상태 변경 알림 채널? (SMS/카카오/이메일/앱푸시)
- 정기구독·반복결제 지원 여부?

**상품·재고**:
- 상품 옵션 조합 방식? (단일옵션/조합옵션/커스텀)
- 재고 관리 시스템(WMS) 연동 여부?
- 상품 노출 규칙? (신상품/인기순/MD추천/개인화)
- 대량 상품 등록 방식? (엑셀/API/크롤링)

**회원·마케팅**:
- 회원 등급 체계? (등급 수, 승급 기준, 혜택 차등)
- 쿠폰·적립금 정책? (발급 조건, 사용 제한, 유효기간)
- 장바구니 이탈 리마케팅 여부?

### 관광 / 지역 포털

**콘텐츠**:
- 관광지 데이터 원천? (자체DB/공공API/외부제휴)
- 다국어 콘텐츠 관리 방식? (번역 주기, 담당자, 품질 검수)
- 계절·시기별 콘텐츠 교체 주기?
- 사용자 생성 콘텐츠(UGC) 수용 범위? (리뷰/사진/코스추천)

**지도·경로**:
- 지도 API? (Kakao/Naver/Google — 라이선스 확인)
- 경로 안내 연동? (도보/대중교통/자가용)
- 오프라인 지도 지원 필요성?

**예약·연동**:
- 숙박·체험 예약 시스템 연동? (자체/외부 위젯)
- 날씨 API 연동 여부?
- 교통 실시간 정보 연동?

### 공공기관

**법적 요건**:
- 웹 접근성 인증 등급 목표? (AA/AAA)
- 개인정보 보호법 적용 범위? (수집 항목, 위탁, 파기)
- 전자정부 표준프레임워크 적용 의무?
- 국가정보자원관리원 보안 기준 적용?

**시스템 연동**:
- 행정정보 공동이용 연동? (주민등록/사업자/부동산)
- 본인인증 방식? (공동인증서/간편인증/SMS)
- 기존 레거시 시스템 현황? (연동 vs 교체)

**거버넌스**:
- 콘텐츠 승인 워크플로? (기안→검토→승인 단계 수)
- 부서별 권한 분리 요건?
- 감사 로그 보존 기간?

### 의료 / 헬스케어

**규제**:
- 의료법 비급여 항목 표시 규정 준수?
- 의료광고 사전심의 대상 범위?
- 진료 예약 시 개인정보(진료과·증상) 수집 동의 절차?
- EMR/HIS 연동 범위? (예약만/진료기록/처방)

**환자 경험**:
- 비대면 진료 지원 여부?
- 진료 대기 현황 실시간 표시?
- 검사 결과 조회 범위?

### 금융 / 핀테크

**보안**:
- 금융보안원 전자금융감독규정 적용?
- 이상거래탐지(FDS) 연동?
- 세션 타임아웃 정책? (자동 로그아웃 시간)
- 키보드 보안/화면 캡처 방지 요구?

**상품·서비스**:
- 금융 상품 비교 기준? (금리/수수료/한도)
- 시뮬레이션·계산기 필요 여부?
- 마이데이터 API 연동?

### IT / SaaS

**온보딩**:
- 프리 티어 제한 범위? (기간/기능/용량)
- 셀프 온보딩 vs 세일즈 데모?
- API 문서 자동 생성 여부? (Swagger/Postman)

**과금**:
- 과금 모델? (구독/종량/하이브리드)
- 결제 주기? (월/연/커스텀)
- 인보이스 자동 발행?
