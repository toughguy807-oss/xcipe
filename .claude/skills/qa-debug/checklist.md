## 품질 기준
- [ ] Iron Law 준수 — 원인 확인 없이 수정 시도 0건
- [ ] 단일 변경 원칙 — 한 번에 1가지만 변경
- [ ] 수정 후 재현 확인 완료
- [ ] 사이드이펙트 확인 완료
- [ ] 추측 표현 0건 ("아마", "것 같다" 등 Iron Law 위반)
- [ ] 3회 실패 시 아키텍처 재검토 제안 여부

## 금지 사항
- [ ] 원인 분석 없는 수정 시도 금지
- [ ] 복합 변경 금지 (여러 개 동시 수정)
- [ ] 검증 없는 수정 완료 선언 금지
