# 복잡도 판단 가이드

> plan-fn 스킬 참조용. SKILL.md §복잡도 판단 기준을 보완하는 상세 체크리스트 + 경계 사례.

---

## 1. 복잡도 3단계 판단 체크리스트

### 높음 (4-탭 구조: 처리+검증+에러+비동기)

아래 **3개 이상** 해당 시 → 높음:

- [ ] 외부 시스템 API 연동 (PG, 지도, 인증, 메일 등)
- [ ] 비동기 처리 (큐, 웹소켓, SSE, 폴링)
- [ ] 복잡한 상태 머신 (3개 이상 상태 전이)
- [ ] 트랜잭션 관리 (다중 테이블 원자적 업데이트)
- [ ] 실시간 동시성 제어 (재고 차감, 좌석 예약 등)
- [ ] 복합 권한 체계 (역할 3개+, 조건부 접근)
- [ ] 다단계 유효성 검증 (클라이언트+서버+비즈니스 룰)
- [ ] 파일 처리 (업로드/변환/썸네일/스트리밍)
- [ ] 복잡한 계산 로직 (할인 중첩, 세금, 환율, 적립금)
- [ ] 다국어/다통화 분기

### 중간 (처리+에러 분기)

아래 **2~3개** 해당 시 → 중간:

- [ ] 단일 외부 API 호출 (단방향, 동기)
- [ ] CRUD + 비즈니스 규칙 (정렬/필터/페이징 + 조건부 로직)
- [ ] 폼 검증 + 서버 검증 (2단계)
- [ ] 조건부 UI 렌더링 (상태 2~3개 분기)
- [ ] 단순 권한 체크 (로그인/비로그인, 관리자/일반)
- [ ] 캐시 전략 (TTL 기반 단순 캐싱)

### 낮음 (서술형)

아래에 **모두** 해당 시 → 낮음:

- [ ] 단순 CRUD (생성/조회/수정/삭제)
- [ ] 외부 연동 없음
- [ ] 분기 로직 1~2개 이하
- [ ] 정적 콘텐츠 표시
- [ ] 단일 권한 레벨

## 2. 경계 사례 판단 가이드

### 낮음 vs 중간 경계

| 사례 | 판단 | 근거 |
|------|------|------|
| 게시판 목록 (페이징만) | 낮음 | 단순 CRUD + 페이징은 ORM 기본 기능 |
| 게시판 목록 (검색+필터+정렬) | **중간** | 복합 쿼리 조건 + 인덱스 전략 필요 |
| 정적 페이지 표시 | 낮음 | DB 조회 1회, 분기 없음 |
| 정적 페이지 + A/B 테스트 | **중간** | 분기 로직 + 트래킹 연동 |
| 이미지 갤러리 (표시만) | 낮음 | 목록 조회 + 라이트박스 |
| 이미지 갤러리 (업로드+리사이즈) | **중간** | 파일 처리 + 검증 |

### 중간 vs 높음 경계

| 사례 | 판단 | 근거 |
|------|------|------|
| 회원가입 (이메일 인증) | 중간 | 검증 2단계지만 상태 전이 단순 |
| 회원가입 (소셜+이메일+본인인증) | **높음** | 3개 외부 연동 + 분기 복잡 |
| 장바구니 (추가/삭제) | 중간 | 세션/DB CRUD + 수량 검증 |
| 장바구니 (쿠폰+적립금+배송비 계산) | **높음** | 복합 할인 규칙 + 동시성 |
| 검색 (DB LIKE 쿼리) | 중간 | 단순 쿼리 + 결과 표시 |
| 검색 (Elasticsearch + 자동완성) | **높음** | 외부 엔진 + 실시간 인덱싱 |
| 알림 (이메일 1건 발송) | 중간 | 단일 API 호출 + 에러 처리 |
| 알림 (멀티채널 + 스케줄링 + 재시도) | **높음** | 큐 + 다중 연동 + 상태 관리 |

## 3. FN 카드 작성 심화 가이드

### 기획근거 작성 패턴

| 소스 | 패턴 | 예시 |
|------|------|------|
| FR 직접 참조 | "FR-###의 AC #{n}에 의거" | "FR-005의 AC #3(중복 이메일 실시간 검증)에 의거" |
| NFR 참조 | "NFR-P-###에 의거" | "NFR-P-001(LCP ≤ 2.5s)에 의거하여 이미지 lazy loading 적용" |
| 벤치마킹 | "경쟁사 {사이트} 분석 결과" | "경쟁사 A사 분석 결과 인기순 정렬이 전환율 23% 높음" |
| 사용자 시나리오 | "페르소나 {이름}의 시나리오 #{n}" | "페르소나 '김과장'의 시나리오 #2(출장 중 모바일 결재)" |

### 기술 노트 4필드 예시

```
## 기술 노트
- 기술선택: Kakao Maps API v3 + GeoJSON 클러스터링
- 대안검토: Google Maps (비용 3배), Naver Maps (커스터마이즈 제한)
- 선택사유: 국내 지도 정확도, 무료 쿼터 30만건/월, 한글 문서 풍부
- 제약사항: 일 쿼터 초과 시 fallback 필요, 해외 커버리지 부족
```

## 4. 업종별 고빈도 고복잡 FN

### 이커머스
- 결제 처리 (PG 연동 + 재시도 + 부분취소)
- 재고 동시성 제어 (낙관적 잠금 + 카운터)
- 할인 엔진 (쿠폰 중첩 + 등급 할인 + 프로모션)

### 관광
- 지도 기반 검색 (위치 클러스터링 + 필터 + 경로)
- 다국어 콘텐츠 동기화 (4개 언어 + 번역 상태 관리)

### 공공
- 전자문서 워크플로 (기안→검토→승인 + 위임 + 반려)
- 본인인증 체인 (공동인증서 + 간편인증 + SMS)

### 의료
- 진료 예약 (의사 스케줄 + 중복 방지 + 대기열)

### 금융
- 거래 처리 (이중 원장 + FDS + 한도 체크)

## 5. 대규모 모드 판단 기준

| 기준 | 일반 모드 | 대규모 모드 |
|------|----------|-----------|
| FR 수 | ≤ 15 | > 15 |
| 높음 복잡도 비율 | ≤ 30% | > 30% |
| 도메인 영역 수 | ≤ 3 | > 3 |
| 외부 연동 수 | ≤ 2 | > 2 |
| 예상 FN 수 | ≤ 25 | > 25 |

대규모 모드 진입 시:
1. Epic 단위 그룹핑 (도메인 기준)
2. Feature 단위 FN 작성 (Epic 하위)
3. Story 단위 세분화 (높음 복잡도 FN만)

---

## 사용법

이 파일은 plan-fn 스킬이 복잡도를 판단하고 FN 카드를 작성할 때 참조합니다.
SKILL.md의 복잡도 3단계 기준이 경계에서 애매할 때, 이 파일의 체크리스트와 경계 사례를 참고합니다.
