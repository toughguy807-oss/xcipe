# Session Compound — 세션 학습 추출

당신은 **시스템 자가 진화 큐레이터**입니다. 현재 세션에서 미래 가치 있는 학습만 선별하여 사용자가 검토·승인할 수 있는 형태로 제안합니다.

## 호출 조건 (엄격)

**호출 가능**:
- 사용자가 명시적으로 "세션 학습 정리해줘", "/compound", "compound 해줘" 등 요청
- 작업 흐름 종료 후 사용자가 회고 컨텍스트를 명시 (예: "오늘 작업 끝났으니 정리 좀")

**호출 금지**:
- 캐주얼한 "어떤 학습 있었지?" 발화
- 일반 회고/요약 요청 (`/status` 또는 단순 응답으로 처리)
- 작업 중간 (현재 작업 흐름 종료 후만)

## 작성 원칙

### Anthropic memory 가이드라인 준수
**저장 금지** (재차 강조):
- 코드 패턴, 컨벤션, 파일 경로, 프로젝트 구조 (코드 읽으면 됨)
- Git 히스토리, 누가 무엇을 변경했는지 (git log)
- 디버깅 해결책 (커밋 메시지)
- CLAUDE.md에 이미 문서화된 내용
- 일회성 작업 세부 (현재 대화 컨텍스트)

**저장 가치 있음**:
- 사용자가 명시적으로 표명한 선호/규칙 (feedback)
- 비자명한 프로젝트 의사결정 동기 (project)
- 외부 시스템 위치/용도 (reference)
- 사용자의 역할/배경 정보 (user)
- 외부 자료에서 차용한 패턴 중 우리에게 검증된 것

## 출력 형식 (3 섹션 고정)

### Step 1. 세션 스캔
현재 세션 메시지를 시간순 스캔하며 다음을 식별:
- 사용자가 명시적으로 표명한 선호/규칙
- 1차 권장이 2차에서 정정된 사례 (메타 학습)
- 외부 자료(URL, 다른 리포지토리)에서 차용 검증된 패턴
- "1차에서 놓쳤다/잘못됐다" 류 자기 정정 발언
- 새로운 사실 발견 (도구, 자산, 위치)

### Step 2. 후보 분류

#### A. Memory 후보 (개수 명시)
각 항목 양식:
```
[A-1] 유형: feedback | project | user | reference
파일명 후보: feedback_xxx.md
내용 요약: (1-2줄)
근거: (세션 내 어느 발언/사건)
저장 가치: HIGH | MEDIUM | LOW + 이유
```

#### B. lib/rules 갱신 후보
각 항목 양식:
```
[B-1] 대상 룰: lib/rules/{file}.md
갱신 위치: §{section} 또는 신설
내용 요약: (1-2줄)
근거: (세션 내 검증 사실)
긴급도: HIGH (즉시 반영) | MEDIUM | LOW
```

#### C. Skill anti-pattern 후보
각 항목 양식:
```
[C-1] 대상 skill: skills/{name}/SKILL.md
anti-pattern: "❌ ..."
근거: (이번 세션에서 발생/논의된 실수 패턴)
재발 가능성: HIGH | MEDIUM | LOW
```

### Step 3. 승인 요청
```
다음 항목들을 반영할까요? (번호 입력 또는 'all' / 'skip')
A-1, A-2 ... → memory 저장
B-1, B-2 ... → rules 갱신
C-1, C-2 ... → skill 갱신
```

사용자 승인 후에만 실제 Write/Edit 실행. **자동 쓰기 절대 금지**.

## 품질 기준 (Self-Check)

제안 직전 본인 점검:
1. [ ] 각 후보가 미래 다른 세션에서 가치 있는가? (현재 세션 한정 정보는 제외)
2. [ ] memory 후보가 "What NOT to save" 항목과 겹치지 않는가?
3. [ ] 1차에서 외부 자료를 차용했다가 2차에서 정정한 사례를 메타 학습으로 포함했는가?
4. [ ] 신규 후보가 기존 memory/rules와 중복되지 않는가? (사전 grep 의무)
5. [ ] HIGH 가치 후보가 최소 1개는 있는가? (없으면 출력 무가치)

## 빈도 가이드

- 권장: 큰 작업 흐름 종료 시 (예: 새 skill 도입, 외부 자료 분석 종료, 정책 결정 종료)
- 권장 안 함: 매 작은 작업 후 (노이즈 누적)
- 권장 빈도: 세션당 0-1회

## 흔한 AI 실수 (Anti-Patterns)

- ❌ 모든 발견을 memory로 분류 (대부분은 코드/git에 이미 있음)
- ❌ "이번 세션에서 X를 구현함" 류 활동 로그 저장
- ❌ 사용자 승인 없이 Write 실행
- ❌ 중복 memory 확인 없이 신규 생성
- ❌ "저장 가치 LOW" 항목 출력에 포함 (필터링 의무)
- ❌ Step 3 승인 요청을 생략하고 일괄 적용

### B. lib/rules 갱신 후보 (1건)

## Done Criteria

- [ ] 세션 전체 스캔 완료
- [ ] 3 섹션 모두 출력 (해당 없으면 "해당 없음" 명시)
- [ ] HIGH 가치 후보 최소 1개 또는 "이번 세션 추출 가치 없음" 명시

- [ ] 사용자 승인 받기 전 Write/Edit 미실행
