# Planning 코어 스킬 실행 패턴

> 6개 Planning 코어 스킬(QST, REQ, FN, IA, WBS, SB)의 실행 빈도, 트리거, 입출력 체인, 갱신 주기를 정리한 참조 문서.

---

## 1. 실행 체인

```
QST → REQ → FN → IA → WBS → SB
              └─→ Dashboard (선택)
```

각 화살표는 선행 산출물이 후행 스킬의 필수 입력임을 의미합니다.

---

## 2. 스킬별 실행 패턴

### plan-qst (고객질의서)

| 항목 | 값 |
|------|---|
| 트리거 | 프로젝트 착수, 고객 요구사항 불명확 시 |
| 빈도 | 프로젝트당 1회 (추가 질의 시 증분 갱신) |
| 입력 | PROJECT.md, input/ 참고자료 |
| 출력 | QST_{코드}_v{n}.md |
| 갱신 조건 | 고객 답변 수신 후 [확인/보류/내부결정] 상태 업데이트 |

### plan-req (요구사항정의서)

| 항목 | 값 |
|------|---|
| 트리거 | QST 완료 또는 요구사항 직접 입력 |
| 빈도 | 프로젝트당 1회 + CR(변경요청) 시 갱신 |
| 입력 | QST 산출물 (연계 모드), 프로젝트 정보 (독립 모드) |
| 출력 | REQ_{코드}_v{n}.md |
| 갱신 조건 | 범위 변경, 우선순위 재조정, NFR 목표값 변경 |

### plan-fn (기능정의서)

| 항목 | 값 |
|------|---|
| 트리거 | REQ 완료 |
| 빈도 | 프로젝트당 1회 + 기능 추가/변경 시 갱신 |
| 입력 | REQ 산출물 (FR/NFR 목록) |
| 출력 | FN_{코드}_v{n}.md |
| 갱신 조건 | 새 FR 추가, 복잡도 재평가, 기술 스택 변경 |

### plan-ia (정보구조설계)

| 항목 | 값 |
|------|---|
| 트리거 | FN 완료 또는 페이지 구조 변경 필요 시 |
| 빈도 | 프로젝트당 1회 (구조 변경 시 갱신) |
| 입력 | FN 산출물, REQ (참고) |
| 출력 | IA_{코드}_v{n}.md |
| 갱신 조건 | 페이지 추가/삭제, 네비게이션 변경, URL 정책 변경 |

### plan-wbs (작업분해구조)

| 항목 | 값 |
|------|---|
| 트리거 | FN 완료 + 일정 산정 필요 시 |
| 빈도 | 프로젝트당 1회 + 범위 변경 시 재산정 |
| 입력 | FN 산출물 (복잡도 정보), REQ (우선순위) |
| 출력 | WBS_{코드}_v{n}.md |
| 갱신 조건 | FN 추가/삭제, 리소스 변동, 마일스톤 변경 |

### plan-sb (스토리보드)

| 항목 | 값 |
|------|---|
| 트리거 | FN + IA 완료 후 데이터 입력 |
| 빈도 | 데이터(JSON) 변경 시마다 재생성 |
| 입력 | data.json (구조화 데이터) |
| 출력 | output/ 내 HTML 파일 |
| 갱신 조건 | 콘텐츠 변경, 페이지 추가, 레이아웃 수정 |

---

## 3. 운영 모드별 실행 빈도

| 스킬 | 신규 구축 | 전환/리뉴얼 | 운영 수정 |
|------|----------|-----------|----------|
| QST | 1회 필수 | 1회 필수 | 선택 (범위 불명확 시) |
| REQ | 1회 필수 | 1회 필수 | CR 단위 갱신 |
| FN | 1회 필수 | 1회 필수 | 변경분만 증분 |
| IA | 1회 필수 | 1회 필수 | 구조 변경 시만 |
| WBS | 1회 필수 | 1회 필수 | 범위 클 때만 |
| SB | N회 (데이터별) | N회 | 변경분만 |

---

## 4. 갱신 연쇄 규칙

선행 산출물 변경 시 후행 산출물의 갱신 필요 여부:

```
QST 변경 → REQ 영향 확인 → FN 영향 확인 → IA/WBS 영향 확인
```

| 변경 유형 | 영향 범위 | 대응 |
|----------|----------|------|
| QST 답변 변경 | REQ ~ WBS 전체 | 변경 FR 식별 → 해당 FN만 갱신 |
| FR 추가 | FN ~ WBS | 신규 FN 추가 → IA/WBS 반영 |
| FR 삭제 | FN ~ WBS | 대응 FN 삭제 → IA/WBS 반영 |
| 복잡도 변경 | WBS만 | 공수 재산정 |
| 페이지 추가/삭제 | IA ~ WBS | IA 갱신 → WBS 반영 |
| 우선순위 변경 | WBS만 | Wave 재배분 |

---

## 5. 독립 실행 vs 연계 실행

| 실행 모드 | 설명 | 추적성 |
|----------|------|--------|
| 연계 | 파이프라인 순서대로 실행 (QST→REQ→FN→...) | FR→FN→UI→TC 전수 추적 |
| 독립 | 단일 스킬만 직접 호출 (`/plan-fn`) | 자체 ID만 부여, 역방향 추적 생략 |
| 부분 연계 | 중간부터 시작 (REQ→FN만) | 입력된 선행물만 참조 |

독립 실행 시에도 산출물 품질은 동일하게 유지합니다. 차이는 추적성 범위뿐입니다.

---

## 사용법

이 파일은 오케스트레이터가 파이프라인 실행 순서를 결정하거나, 변경 영향 범위를 판단할 때 참조합니다.
개별 스킬의 상세 입출력 명세는 각 SKILL.md를 참조하십시오.
