1. plan-sb 개요

plan-sb는 SYS v4 기획 패키지의 화면설계서(SB) 자동 생성 스킬입니다. JSON 데이터로부터 HTML/PDF 화면설계서를 생성하고, 선택적으로 Figma 공식 Remote MCP를 통해 Figma 디자인 파일까지 자동 변환합니다.

핵심 가치

항목설명
입력 자동 감지FN 산출물 또는 URL/캡쳐/자연어 발화 → 모드 자동 결정 (Mode A/B/C, 연계/독립 분기)
다중 프레임 타입Design · Description · MSG Case · Component Guide 4종 지원
테마 시스템default · CAFE_ONEPAGE · STOCK · TEST-V21 · VALIDATE_L2 등 5종 프리셋
스키마 v2.1containerType + appearance 속성 + msgCase groupDescription/groupFlow
Figma ExportF1~F7 플레이북 — 공식 Remote MCP 기반 (plugin 불필요)

입출력

입력

  • FN_*.md 산출물 (연계 모드)
  • URL / 도메인 (Mode B)
  • input/*.png 캡쳐 (Mode B/C)
  • input/*.json 기존 SB (Mode C)
  • 자연어 발화 (Mode A)

출력

  • output/{name}/{date}/{prefix}.html
  • output/{name}/{date}/{prefix}.pdf
  • (선택) _figma/{prefix}-figma.html
  • (선택) Figma 파일 (1920×1080 슬라이드)

3. 입력 감지 + 모드 자동 분류

3.1 Step 0-Init Welcome — 단독 패키지 첫 실행 4 옵션

input/output/JSON 모두 비어있을 때 자동 안내. 사용자가 옵션을 선택하지 않아도 자연어 발화로 모드가 자동 결정됨.

옵션모드트리거첫 사용자 응답 예시
A 신규 화면 설계Mode AHTML 목업 → SB"쇼핑몰 상품상세 새로 만들어줘"
B 운영 수정Mode B현행 사이트 캡쳐 + 변경 마커"https://example.com 메인 배너 위치 변경"
C 기존 SB JSON 수정Mode Cinput/*.json 발견 시 자동"input/old-sb.json 슬라이드 3 수정"
D PDF/PPT 포맷 참고Mode A/Cinput/*.pdf 분석 → 테마 자동 생성"이 포맷대로 신규 SB 만들어줘"

3.2 Mode 자동 분류 — 우선순위 5단계

사용자 발화 + 입력 자원을 분석해 모드를 즉시 결정. 조건 명확 시 사용자 확인 생략.

#조건자동 결정사용자 확인마커 출력
1URL/도메인 포함Mode B + visit.js 자동생략[모드 자동 결정] Mode B (근거: URL제공)
2input/*.png 캡쳐 존재Mode B생략[모드 자동 결정] Mode B (근거: 캡쳐존재)
3"수정/변경/운영/현행/추가/as-is/to-be" 매칭 + 캡쳐 없음Mode B 강제 + URL 1회 질의URL 미제공 시만[모드 자동 결정] Mode B (근거: 키워드)
4"신규/새로/만들어/구축" 매칭 + 캡쳐 없음Mode A (HTML 목업)생략[모드 자동 결정] Mode A (근거: 키워드)
5모호 (리뉴얼/개편/혼합/충돌)Mode C 추정 자동 진행묻되 자동 진행[모드 자동 추정 — 모호] Mode C
핵심 정책: 모호 케이스에서도 사용자 응답 대기 없이 즉시 다음 Step 진행. 도중 "Mode X로" 입력 시 영향받는 산출물만 재생성.

분류 로직 (의사코드)

hasUrl    = 발화에 http/https/도메인 패턴
hasCapture = input/*.{png,jpg,jpeg} 존재
isOps     = "수정|변경|운영|현행|추가|as-is|to-be" 매칭
isNew     = "신규|새로|만들어|구축" 매칭
isMixed   = "리뉴얼|개편" or (isOps && isNew)

if (hasUrl)              → Mode B 자동 + visit.js
elif (hasCapture)         → Mode B 자동
elif (isOps && !hasCapture) → Mode B 강제 + URL 1회 질의
elif (isNew && !hasCapture) → Mode A 자동
elif (isMixed)            → Mode C 추정 자동 진행
else                      → Mode A 추정 (가장 보수적 기본값)

3.3 연계 vs 독립 모드 (FN 입력 감지)

Mode A/B/C와 직교(orthogonal)된 별도 축. FN 파일 존재 여부로 분기.

모드조건동작fnRef추적성
연계 모드output/{name}/*/FN_*.md 존재FN 기반 screens 자동 구성, AC가 Description에 자동 반영✅ 매핑FR→FN→SB ID 체인 유지
독립 모드FN 미존재, 프롬프트/이미지/URL만 입력프롬프트 기반 직접 구성❌ 생략역참조 없음, 프로토타이핑 적합

3.4 모드 조합 매트릭스 (Mode × FN — 6 케이스)

Mode A 신규Mode B 운영Mode C 혼합
연계 (FN 있음)FN → 신규 SBFN → 현행 비교 SBFN → 리뉴얼 SB
독립 (FN 없음)프롬프트 → SBURL → 캡쳐 SBURL + 프롬프트 → 혼합 SB

판정 출력 예시

[입력 감지] FN: 3건 발견 → 연계 모드
[모드 자동 결정] Mode A (근거: "새로 만들어줘" 키워드 + 캡쳐 없음) — 사용자 확인 생략
[Step 0-Init 환경 점검] 1234 ms — 5/5 통과
→ Step 0.5 기획 확정 게이트로 진행

4. Step별 상세

0iStep 0-Init: Welcome + 환경 점검

SB 스킬 트리거 즉시 preflight.js를 1회 실행하여 환경을 빠르게 진단.

5개 검사 항목 (각 1.5초 timeout)

#항목통과 조건실패 시 영향
1node-versionNode 18+전체 실행 불가
2skill-mdSKILL.md 존재 + ≥1KB플러그인 미설치
3playwright-pkgnode_modules 또는 글로벌Mode B 불가
4chromium-binaryms-playwright 캐시 폴더 존재Mode B 캡쳐 불가
5visit-jsscripts/visit.js syntax OK자동 캡쳐 불가
처리 정책: exit 0 → 정상 진입 / exit 2 → 가이드 출력 + 사용자 응답 대기 / 5초 초과 → "플러그인 미정상 로드" 안내.

0pStep 0-Pre: visit.js 셋업 게이트 Mode B/C 필수

현행 사이트 URL 제공 시 visit.js 실행 결과를 반드시 검증. 실패 시 추정 진행 차단.

게이트 절차

  1. node scripts/visit.js {URL} 실행 → exit code = 0 검증
  2. exit code 2 (SETUP_FAIL) → playwright/chromium 미설치 또는 사내망 차단
  3. input/screenshot.png 존재 + 크기 > 10KB
  4. input/structure.json 존재 + parseable
금지 사항: visit.js 실패 후 "사이트 구조 추정 SB 작성" 절대 금지. exit code 0 아니면 후속 Step 진입 차단.

0Step 0: 입력 감지 (FN 연계 / 독립)

모든 모드 공통. FN 파일 자동 스캔으로 연계/독립 결정.

출력 마커

[입력 감지] FN: 3건 발견 → 연계 모드
[입력 감지] FN: 미발견 → 독립 모드

0.5Step 0.5: 기획 확정 게이트

JSON 생성 전 화면 구성/페르소나/콘셉트 사용자 합의 필수. "확정?" 사용자 응답 필요.

합의 항목

  • 대상 페이지/화면 목록 (전체 슬라이드 수)
  • 페르소나/타겟 유저
  • 핵심 콘셉트/톤앤매너

items[].category 7종 (2026-05-11 신설)

items[]에 category 키를 추가해 기능 동작 의미를 분류. CTA/액션 요소는 verify.js가 누락 자동 감지.

#category용도예시
1trigger발동 조건 (디바이스별 차이 포함)Click (PC) / Tap (Mobile) / Enter 키
2enable_cond활성/비활성 조건필수 필드 입력 + 약관 동의 시 활성, 그 외 disabled
3action실행 동작 (API/이동/모달)POST /api/order/checkout
4success성공 처리/order/complete 이동 + 토스트
5failure실패 처리 (4xx/5xx/network)4xx 인라인 / 5xx 모달 / 401 → /login
6state수량/조건 상태 분기1개일 때 화살표 미노출 / 2개+ 노출
7permission권한 분기비로그인 → /login / VIP → 특가 뱃지
제외 영역 (시각 = plan-sb 책임 X): 픽셀값(1920×600) / 그리드 컬럼 수 / spacing 토큰 / 색상 / 폰트 크기 → design-layout + publish-style + design-knowledge 영역

verify.js 자동 감지 (Phase 3.3)

  • 액션 키워드(버튼/CTA/결제/로그인/제출/구매/검색 등 25종) 매칭 + items[] category 0건 → WARN-PRE

2Step 2: generate.js 실행

data.json + theme + template.html을 결합하여 HTML/PDF 생성.

node .claude/skills/plan-sb/scripts/generate.js \
  --data path/to/data.json \
  --theme themes/default.json \
  --output output/{name}/{date}/{prefix}.html

처리 흐름

F6 detach JavaScript (use_figma)

const master = await figma.getNodeByIdAsync(MASTER_ID);
const body = master.children[0];
const slides = body.children.slice();
const TARGET_W = 1920, GAP = 40;
const page = body.parent.parent;

for (let i = 0; i < slides.length; i++) {
  const s = slides[i];
  page.appendChild(s);
  if (Math.abs(s.width - TARGET_W) > 1) s.rescale(TARGET_W / s.width);
  s.x = i * (TARGET_W + GAP);
  s.y = 0;
  s.name = `Slide ${i + 1} - ${slideTypes[i] || ""}`;
}
if (body.children.length === 0) body.remove();
if (master.children.length === 0) master.remove();
