ELUO Design Sync

Claude Code에서 만든 디자인을 Figma로 보내고, 디자이너가 수정한 걸 다시 코드로 가져오는 구조.

01 이게 뭔지

한 줄 요약: Claude Code ↔ Figma를 연결해서 디자인 토큰 + 레이아웃을 양방향으로 동기화하는 것.

Claude Code코드 생성
HTML/CSS 만듦
↔
CLI (CDP)figma-use
자동 연결
↔
Figma Desktop디자이너 작업
디자인 수정
비유: Google Docs처럼 코드와 Figma가 같은 디자인 시스템을 공유합니다. 한쪽이 바꾸면 다른 쪽에서 가져올 수 있습니다. 단, 실시간 동기화는 아니고 "가져와줘"라고 하면 동기화됩니다.

왜 필요한지

지금 (연결 없음)연결 후
Claude Code가 HTML 만듦 → 디자이너한테 캡처 보내줌 → 디자이너가 Figma에서 처음부터 다시 그림Claude Code가 HTML 만듦 → Figma에 자동으로 프레임 생성 → 디자이너가 보정만
디자이너가 수정함 → 개발자한테 구두로 "색상 바꿨어요" → 수동 반영디자이너가 수정함 → "가져와줘" 한마디 → 변경 자동 감지 + 코드 반영
토큰(색상/폰트/간격)이 코드와 Figma에서 따로 관리 → 불일치하나의 토큰 JSON이 양쪽을 지배 → 항상 일치

핵심 도구 4개

도구역할비유
Figma 공식 MCP (권장)Claude Code → Figma 양방향 통신. 노드 CRUD, Variables, 컴포넌트, Auto Layout 모두 지원Figma의 공식 리모컨
figma-cli (fallback)MCP가 안 되는 경우 보조. Variables 대량 import 등예비 리모컨
design-knowledge (스킬)외부 디자인 시스템(KDS 등)의 토큰을 파이프라인에 주입재료 주입기
changelog + baseline변경 이력 추적 + diff 비교 기준 (우리만의 차별점)변경 일지

전환 결정 (2026.4): figma-cli + eval bridge → Figma 공식 MCP로 전환. 2026.3.24 Figma가 공식 MCP 베타 출시. Claude Code 네이티브 지원.

02 어떻게 되는지

CLI가 Figma Desktop에 직접 연결하는 원리. Plugin 설치 없이 동작.

연결 원리 (MCP 기반)

Figma Desktop이 내장 MCP 서버(localhost:3845)를 노출합니다. Claude Code가 MCP 프로토콜로 직접 통신합니다. 별도 플러그인 설치 불필요.

동작 원리
Claude Code가 사용자 요청 받음 ("이 토큰 Figma에 푸시")
       │
       ▼
MCP 프로토콜로 Figma Desktop 호출 (http://127.0.0.1:3845/mcp)
       │
       ▼
Figma가 use_figma 도구 실행 (Variables 생성, 노드 CRUD)
       │
       ▼
결과 + 노드 ID/속성 응답
       │
       ▼
baseline.json 갱신 + changelog.json 자동 기록
비유: 이전(figma-cli)이 카페 주방에 직접 들어가서 음식 만드는 거였다면, MCP는 정식 주문 창구로 시키는 것. 안전하고 표준화됨.

설정 방법

  1. Figma Desktop 최신 버전 → 디자인 파일 열기
  2. Shift+D → Dev Mode 전환
  3. Inspect 패널 → Enable desktop MCP server 클릭
  4. ~/.claude.json 프로젝트 섹션의 mcpServers에 figma-desktop 추가
  5. Claude Code 재시작 → /mcp로 연결 확인

역할 분담: 누가 뭘 하는지

하는 일담당왜 이걸로
토큰을 Figma에 쓰기 (Push)CLIFigma에 쓰기는 CLI만 가능 (MCP는 읽기만)
레이아웃을 Figma에 생성 (Push)CLIFrame, Text, Image 생성 = CLI
토큰 현재값 읽기 (Pull)CLIVariables는 MCP로 못 읽음 (Pro 플랜)
노드 구조 읽기 (Pull)MCP이미 설치됨, URL 주면 바로 읽기
텍스트/이미지 읽기 (Pull)MCP위와 동일
HTML/CSS 코드 생성Claude Code 스킬publish-markup, publish-style

03 Push — 코드 → Figma로 보내기

Claude Code가 만든 것을 Figma에 자동 생성. 사용자 액션 0.

보낼 수 있는 것

항목가능예시
색상 토큰Oprimary-500: #EF151E → Figma Variable
폰트 스타일OPretendard 24px Bold → Figma Text Style
간격/radiusOspacing-md: 16, radius-sm: 4 → Figma Variable
그림자Oelevation-1 → Figma Effect Style
프레임 구조 (Flex)Odiv.flex-row → Frame auto-layout
프레임 구조 (Grid)Odiv.grid-3cols → Frame grid layout
텍스트O"로그인" → Text 노드
아이콘 (표준)O15만개 Iconify (lucide, mdi 등)
커스텀 SVGOVector path로 생성
이미지OURL 참조로 Image Fill
절대 위치Ox, y 좌표 지정

Push 실제 동작

사용자: "KDS로 로그인 페이지 만들어서 Figma에 넣어줘"
# Claude Code가 자동 실행하는 것들:

# 1. 토큰으로 HTML/CSS 먼저 생성
✓ login.html + login.css 생성 완료

# 2. Figma 연결
$ figma-use connect
✓ Figma Desktop 연결됨

# 3. 토큰을 Figma Variables로 생성
✓ color/primary-500: #EF151E
✓ color/gray-900: #191A1B
✓ spacing/md: 16
✓ ... (19개 토큰)

# 4. HTML 구조를 Figma 프레임으로 변환
$ figma-use render login.jsx
✓ Frame "LoginCard" (400x auto, flex-column, gap:16)
✓   Text "로그인" (24px, Pretendard SemiBold)
✓   Frame "EmailInput" (fill x 44, bg:gray-50, radius:4)
✓   Frame "PasswordInput" (fill x 44, bg:gray-50, radius:4)
✓   Frame "LoginButton" (fill x 44, bg:primary-500, radius:4)
✓     Text "로그인" (16px, white)

# 5. baseline 스냅샷 저장
→ .figma-sync/baseline.json (19 tokens + 6 nodes)
→ 디자이너에게 "Figma에 세팅 완료" 알림
결과: 디자이너가 Figma를 열면 로그인 화면의 프레임 구조 + 토큰이 이미 잡혀있음. 처음부터 그리는 게 아니라 보정만 하면 됩니다.

"가져와줘" 한마디면 뭐가 바뀌었는지 감지하고 코드에 반영.

노드 삭제OO — 해당 코드 제거

Pull 실제 동작

사용자: "피그마에서 수정된 거 가져와줘"

# 1. CLI로 현재 토큰값 전수 읽기
$ figma-use export variables --json
→ 19개 토큰 읽기 완료

# 2. baseline과 비교 (diff)
⚠ 토큰 변경 3건 감지:
  color/primary-500:  #EF151E → #E50914
  radius/sm:          4 → 8
  shadow/card:        0 2px 8px → 0 4px 16px

# 3. MCP로 노드 구조 읽기
→ figma_get_file_nodes
⚠ 구조 변경 1건 감지:
  + SocialGroup (children: KakaoBtn, NaverBtn, GoogleBtn)

# 4. 속성 변경 읽기
⚠ 속성 변경 2건:
  Title: font-size 24→28, weight 600→700
  Input: padding 8px→12px

# 5. 코드 반영
✓ CSS Variables 3건 업데이트
✓ 속성 변경 2건 반영
→ 구조 변경 1건: HTML 추가 제안 (확인 필요)

추가된 객체는 어떻게 변환하는지

  layout: VERTICAL, gap: 8
  children:
   INSTANCE "KakaoBtn"
    fill: #FEE500
    text: "카카오 로그인"
   INSTANCE "NaverBtn"
    fill: #03C75A
    text: "네이버 로그인"
   INSTANCE "GoogleBtn"
    fill: #FFFFFF
    text: "Google 로그인"
→
Claude Code가 생성한 코드

<div class="login__social">
  <button class="btn--kakao"
   style="background:#FEE500">
   카카오 로그인
  </button>
  <button class="btn--naver"
   style="background:#03C75A">
   네이버 로그인
  </button>

</div>

06 실제 시나리오: KDS로 로그인 페이지

처음부터 끝까지 전체 흐름

토큰 주입 + 1차 코드 생성 자동design-knowledge에 KDS 토큰 주입 → publish 스킬로 login.html + login.css 자동 생성
Figma에 Push 자동CLI가 Figma Desktop에 연결 → 토큰을 Variables로, HTML 구조를 프레임으로 자동 생성 → baseline 저장
디자이너 작업 디자이너Figma에서 로그인 화면 보정: 색상 미세 조정, radius 변경, 소셜 로그인 버튼 추가, 폰트 크기 변경 등

07 할 수 없는 것

솔직한 한계. 이건 어떤 도구를 써도 동일.

항목왜 안 되는지대신 어떻게
실시간 알림
디자이너가 바꾸는 순간 알림		 Figma가 외부에 변경 이벤트를 안 보냄 "가져와줘"라고 하면 그때 비교
반응형 코드 추출
모바일/데스크톱 분기		 Figma는 고정 크기 프레임만 있음. "화면 줄이면 이렇게 바뀐다"는 정보 자체가 없음 코드에서 직접 반응형 작성 (이건 원래 코드 영역)
인터랙션 / 애니메이션
hover, 클릭, 전환 효과		 Figma 프로토타입 ≠ CSS/JS. 데이터 구조가 다름 코드에서 직접 작성 (이건 원래 코드 영역)
비즈니스 로직
OAuth URL, API 호출 등		 Figma에 이 정보가 없음. 디자인과 무관 사람이 알려줘야 함
요약: Figma에 있는 "보이는 것"(색상, 구조, 텍스트)은 전부 가져옵니다. Figma에 없는 "동작하는 것"(반응형, 인터랙션, API)은 코드에서 합니다. 이건 Figma의 근본적 성격이라 어떤 도구를 써도 동일합니다.
